================================================================================
  💡 What It Does / 核心用途
================================================================================

  1. Cross-Session Context Continuity / 跨会话上下文连续性
  ─────────────────────────────────────────────────────────
     [EN] Auto-detect or manually archive key conversation nodes. On next
     session start, AI automatically offers to restore context — loading a
     condensed version first (≤500 chars) to prevent context bloat.

     [CN] 自动检测或手动存档关键对话节点。下次会话启动时 AI 自动提议恢复上下文，
     优先加载压缩版（≤500字），避免上下文膨胀。


  2. Workflow Classification / 工作流分类管理
  ─────────────────────────────────────────────────────────
     [EN] One subfolder = one independent workflow thread (strict MECE).
     Smart semantic matching auto-identifies where each archive belongs.
     Built-in active → dormant → archived lifecycle management.

     [CN] 一个子文件夹 = 一条独立工作流（严格 MECE 互斥）。智能语义匹配自动识别
     存档归属。内建 active → dormant → archived 生命周期管理。


  3. AI Proactive Detection / AI 自主感知关键节点
  ─────────────────────────────────────────────────────────
     [EN] Framework births, major decisions, milestone completions — AI
     proactively proposes archiving. Deduplication checks, fork detection,
     and index integrity validation run automatically.

     [CN] 框架诞生、重大决策、里程碑完成 → AI 主动提议存档。去重检查、分叉检测、
     索引校验全自动运行。


  4. Dual Write + Auto Validation / 双份写入 + 自动校验
  ─────────────────────────────────────────────────────────
     [EN] Writes to both Claude Code's memory system (auto-loads next session)
     AND a human-readable Desktop folder. Post-write index integrity check
     ensures nothing drifts out of sync.

     [CN] 同时写入 Claude Code 记忆系统（下次会话自动加载）和桌面可翻阅文件夹。
     写入后自动校验索引完整性，防止索引漂移。

================================================================================
  🧠 Design Principles / 设计原理
================================================================================

  First Principles (Aristotle → Musk)
    [EN] Manage attention, not files. The real product is "where was I?"
    [CN] 管理注意力流，而非文件。真正的产品是"上次聊到哪了"。

  MECE (McKinsey / Barbara Minto)
    [EN] One folder = one workflow. Strictly mutually exclusive, collectively exhaustive.
    [CN] 一个子文件夹 = 一条工作流。严格互斥，完整穷尽。

  OODA Loop (John Boyd, US military strategist)
    [EN] Observe → Orient → Decide → Act. Human sits at the Feedback node as final reviewer.
    [CN] 观察→定位→决策→执行。人类在 Feedback 节点做终审。


================================================================================
  🔬 10 Long-Term Risks — Addressed / 10 项长期风险修正
================================================================================

  Simulated across 6 months / 50 workflows / 200 saves. All mitigated in-design.
  经 6 个月 / 50 条工作流 / 200 次存档模拟验证，全部在设计层面修正。

  #1  Index Drift / 索引漂移          → Post-save auto validation + repair
  #2  Context Bloat / 上下文膨胀       → Condensed context after ≥5 saves (≤500 words)
  #3  Ghost Context / 幽灵上下文       → Prompt-before-load with skip button
  #4  Matching Degradation / 匹配退化  → Semantic match % comparison, user picks
  #5  Workflow Proliferation / 泛滥    → active → dormant → archived lifecycle
  #6  No Garbage Collection / 永不删除 → >180-day zombie alert, max 1/session
  #7  Self-Reference Rot / 自引用腐败   → Self-check: skill files = source of truth
  #8  Version Divergence / 版本分歧    → Clear boundary: skill=exec, memory=remind
  #9  Permission Decay / 权限衰减      → Permanent grant declaration embedded
  #10 Session Forking / 会话分叉       → lastUpdated conflict detection + warning
